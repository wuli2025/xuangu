📡 A股舆情情绪分析模块
实施规划

给「智能选股」系统的舆情板块，一份能直接照着干的落地蓝图。所有工具、接口、模型均已联网核实（2026）。

合规优先 行为代理打底 文本情感增强 反向指标逻辑 一周可跑 MVP

0 · 核心设计思路（先定调，这决定了一切）为什么不是"上来就爬股吧"

关键判断：A股舆情不要以"爬虫+文本情感"为主力，要以"合规行为代理指标"为主力。
原因有三：① 爬雪球/东财股吧在 2025 修订的《反不正当竞争法》下有真实民事风险，且爬虫接口三天两头失效、维护成本高；② 散户"行为"（人气榜飙升、关注指数、北向资金）比散户"嘴上说的"更真实、更难造假；③ akshare 已经把东财/雪球/百度的热度数据做成了免费、合规、结构化的接口，直接拿来即可。
三条设计原则：
① 能用官方聚合接口，就不爬虫 —— akshare 的热度/资金接口是主力，爬虫只在"必须要原文"时才上。
② 情绪是反向指标 —— 散户人气爆棚、人人喊涨 = 见顶预警，不是买入理由。模块要输出"过热/过冷"而不是简单"利好/利空"。
③ 先量化热度，再做文本情感 —— 热度指标 1 天就能搭好；文本情感（finbert-tone-chinese）是第二阶段的精度增强，不挡 MVP。

2 · 工具最终选型（我替你拍板）每一项都给"为什么选它"

🧠 文本情感模型 —— 选 yiyanghkust/finbert-tone-chinese

主力推荐 huggingface.co/yiyanghkust/finbert-tone-chinese
为什么是它：目前唯一"中文 + 金融 + 开箱即用"的三分类情感模型（0中立/1正面/2负面），正好对应舆情的看涨/看跌/中性。准确率 0.88、宏 F1 0.87，transformers 几行代码上线。
一个坑要提前知道：它训练语料是"分析师报告"（书面语），而股吧/雪球是口语+emoji+黑话（"梭哈""韭菜""抬轿子"），存在域偏移。上线后建议人工标注几百条股吧样本做一轮微调或阈值校准。
候选定位结论

🗂️ 数据底座 —— akshare（主）+ Tushare（备）

akshare 免费 是 A 股舆情+行情数据的核心，热度/资金/龙虎榜接口齐全且合规。Tushare 部分接口（moneyflow / hsgt_top10 / top_list）需积分，作为备份交叉验证。

🕷️ 爬虫（第二阶段才上）

微博 → MediaCrawler；东财股吧 → EastMoney_Crawler；雪球 → XueQiuSuperSpider。仅在 akshare 热度接口不够、需要原文做细粒度情感时启用。

3 · 「情绪温度计」指标设计把一堆数据变成一个 0–100 的分数 + 反向信号

最终我们要给每只股票（和大盘）算一个 情绪温度 0–100：0=冰点恐慌，100=狂热过热。再叠加"反向解读"——温度太高反而要警惕。

三个分维度（各自归一化到 0–100）

维度由什么算权重建议
热度分 H千股千评关注指数 + 人气榜排名变化 + 雪球讨论量 + 百度热搜（取分位数）40%
资金分 F北向净流入 + 主力净流入 + 龙虎榜净买（标准化）35%
文本情感分 Sfinbert-tone-chinese 对当日股吧/雪球/微博文本的"看涨比例"25%
情绪温度 = 0.40·H + 0.35·F + 0.25·S　（每项先归一化到 0–100）

反向解读规则：
　温度 ≥ 80 且 较 5 日均值飙升 → 🔴 过热预警（散户狂热，警惕回撤）
　温度 ≤ 20 且 持续低迷　　　 → 🟢 冰点信号（恐慌见底，可关注）
　20 < 温度 < 80　　　　　　 → ⚪ 中性区间（结合基本面/技术面）
为什么要"飙升/持续"这个二阶条件？ 单看绝对温度会误判——茅台天天高人气是常态。真正有用的是变化率：情绪从平静突然飙到狂热，才是反转信号。这一步把"水平值"升级成"动量值"。
验证方法（必须做）：拿过去 1–2 年历史数据回测——"温度≥80 的股票，未来 5/10/20 日收益是否真的更差？"。验证通过这个指标才可信，否则就是拍脑袋。（参考 AAII 散户情绪反向指标的研究方法）

5 · 分阶段实施路线图从能跑到好用，三步走

第 1 周 · MVP（零爬虫零风险）
只用 akshare 第①②层接口，搭出「热度分 H + 资金分 F」→ 情绪温度（先不含文本情感）。每日定时抓取入库，输出 JSON + 一个简单温度计页面。目标：今天就能看到每只股票的情绪温度和过热预警。
第 2–3 周 · 文本情感增强
接入 finbert-tone-chinese，对股吧/雪球文本（先用 akshare 能拿到的，必要时上 EastMoney_Crawler）做情感打分，补上 S 维度。人工标注几百条做阈值校准。
第 4 周 · 回测验证 + 反向信号
用 1–2 年历史数据回测"过热→回撤、冰点→反弹"是否成立，标定阈值（80/20）。把验证过的反向信号正式接入。
进阶 · 微调 + 多源 + 接主系统
用熵简 FinBERT 基座做领域微调；扩展微博/抖音/B站；把情绪温度作为"情绪分析师 agent"的事实输入，喂给 AI 大师决策层。
给小白的话：别一开始就追求大而全。第 1 周的 MVP 用纯 akshare 就能产出有用的"情绪温度+过热预警"，先跑起来看到效果，再逐步加文本情感和爬虫。

6 · 输出格式与主系统对接舆情板块吐什么给"AI 大师决策层"

{
  "stock": "600519 贵州茅台",
  "date": "2026-06-09",
  "sentiment_score": 83,          // 情绪温度 0-100
  "level": "过热",                 // 冰点/偏冷/中性/偏热/过热
  "signal": "🔴 过热预警-警惕回撤",  // 反向解读结论
  "breakdown": { "热度H": 88, "资金F": 79, "文本情感S": 80 },
  "evidence": {
    "千股千评关注指数排名": "第12名(↑35)",
    "北向资金": "净流入2.1亿",
    "雪球讨论量": "近7日+210%",
    "股吧看涨比例": "76%"
  },
  "contrarian_note": "散户关注度7日飙升、看涨情绪集中,历史上此状态后20日跑输大盘概率偏高"
}
这段 JSON 就是舆情板块交给"投资委员会"的情报卡。决策层的不同门派 agent 会这样用它：趋势派看到"过热"可能顺势但收紧止损；价值派把它当噪音忽略；反向派直接当卖出预警。同一份情绪数据，不同门派读出不同动作。

⚠ 合规与风险2026 现行法律，务必遵守

爬虫红线（《反不正当竞争法》2025修订第13条 + 个保法 + 数安法）：
① 禁止"避开或破坏技术管理措施"获取数据（微博诉脉脉判例已确立）；② 公开信息处理也须"合理范围、符合原始目的"，股吧/雪球言论含个人信息，批量采集有风险；③ 破坏 robots 协议是侵权认定要素。
降风险做法（按优先级）：① 主用 akshare 聚合接口（公开聚合数据，风险最低，本规划主力）；② 微博走官方开放平台 API；③ 必须爬时——只采公开非敏感内容、遵守 robots、控频不破坏服务、不绕登录、数据仅内部分析不二次售卖；④ 绝不突破反爬、不存可识别个人画像。
投资免责：情绪温度是概率性反向信号，会出错、会早到晚到，不构成投资建议。必须结合基本面/技术面，回测表现不代表未来收益，风险自负。
