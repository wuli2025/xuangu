输入一只股票，得到一份带买卖建议的报告。

PRD v1.0 · 基于 polaris-app 改造 · Tauri + Vue3 + Rust · 零本地模型
00

产品定位

把能跑定时 agent、能切 API 模型的 polaris-app，重做成一个舆情驱动的选股报告终端。

保留母体的自动化引擎和供应商引擎，砍掉内容创作工具与以聊天为中心的交互。用户要么输入自选股拿即时报告，要么让它定时盯盘、自动推送。

01

母体改造

不是从零写，是换壳换大脑。骨架全留，把"写文章"换成"出报告"。

母体模块处置新角色
自动化引擎改造监控引擎核心：定时盯盘、出报告，定时器原样复用
供应商坞扩展API 接口管理：LLM key + 数据源 key 统一管理
对话管线保留报告生成的后台执行引擎（不再呈现为聊天）
人格系统改造门派引擎：9 大投资大师 = 9 套决策逻辑
技能中心 / 推送 / 知识库保留数据采集 skill · 报告推送 · 研究档案
聊天首页降级让位给舆情看板
自媒体 / PPT / 视频 / 建站移除与选股无关，整组下线

04

门派引擎

9 大门派各是一套系统提示。同一份情绪数据，不同门派读出不同动作。

模式表现
单门派全程用该门派逻辑（如趋势派 → 7-8% 止损 + 3:1 盈亏比）
委员会多门派各出意见 + 信心分，汇总投票（仿 ai-hedge-fund）
口吻约束：每条建议都标注「基于某门派逻辑的研究参考，非投资建议」，给理由与风险，绝不打包票。
05

数据源

海外 + 国内双轨，B 站作为国内自媒体情绪重点。

📺 B 站增强

财经 UP 主视频、弹幕、评论 → 国内年轻散户的实时情绪。当 UP 主与弹幕一边倒看多时，作为反向过热预警。弹幕的实时性与情绪浓度是微博、股吧没有的。

轨道来源接入
🌍 海外Reddit · StockTwits · Stocksera 聚合PRAW / 免费 API
🇨🇳 国内B 站 · 雪球 · 东财股吧 · 微博采集 skill / akshare
💸 资金面北向资金 · 龙虎榜 · 内部交易akshare / Stocksera
06

情绪分析

情绪打分交给接入的 API 大模型，不用任何本地模型。

①
取数
文本 · 热度 · 资金
②
API 打分
看涨/看跌/中性 + 理由
③
合成温度
0–100
④
反向解读
过热 / 冰点
情绪温度 = 0.40·热度 + 0.35·资金 + 0.25·文本情感（各项归一化 0–100）。
温度 ≥ 80 且飙升 → 🔴 过热预警；≤ 20 且持续 → 🟢 冰点信号；中间结合基本面。
上线前必须回测：用 1–2 年历史数据验证「过热→回撤、冰点→反弹」是否成立，再标定阈值。

09

里程碑

先跑通一条「输入股票 → 出报告」的链路，再加广度。

M1 · 1–2 周
骨架换壳。重排导航，首页换看板，接通 akshare，看到自选股真实情绪热度。
M2 · 3–4 周
报告跑通。接 API 模型做情绪打分 + 门派提示，输出报告卡并落档。
M3 · 5–6 周
数据广度。接 Stocksera / Reddit 与 B 站采集，三维温度 + 反向解读 + 回测。
M4 · 7–8 周
自动化。内置监控流程 + 微信/飞书推送 + 委员会模式。
