# SENTIO 量化系统 · 工业级改造与优化计划

> 目标定位:把现有「零售级研究工具」升级为「可实盘运维的工业级量化系统」。
> 收益目标的诚实表述:**追求年化 35–55%、夏普 >1.5、最大回撤 <15% 的风险调整后收益**,
> 在该分布下单月 ≥8% 的频率显著上升——而非承诺「稳定月化 8%」(后者在 A 股不可持续,
> 见 `strategy.py:541`,任何承诺稳定月化的都是骗局)。

## 一、现状盘点(已是扎实底子)

| 模块 | 文件 | 评价 |
|---|---|---|
| 情绪采集(热度H+资金F) | `collect.py` | ✅ 零爬虫合规,东财 TLS 坑已解 |
| 横截面多因子 | `strategy.py` | ✅ z-score 复合评分 + 等风险仓位 + 月度回测 |
| 斐波趋势跟踪(事件驱动) | `fib_engine.py` / `fib_scan.py` | ✅ 无未来函数、组合层日频权益、参数寻优。实测年化 33.4%/夏普 1.3 |
| 编排 + 调度 | `run_all.py` / `run_fib.py` / `schedule_daily.ps1` | ✅ 盘后 15:30 自动跑 |

**核心短板(限制收益与"工业级"的 7 个洞):**
1. **宇宙太小且静态**(32 只手选龙头)→ 幸存者偏差 + 趋势信号供给不足。这是收益天花板第一因。
2. **参数寻优在样本内**(`grid_search` 在同一段数据上选最优又报告)→ 过拟合幻觉,无样本外验证。
3. **数据层脆弱**:单一新浪源、每日全量重拉、无增量行情库、无 point-in-time。
4. **回测不含 A 股微观结构**:无 T+1、无涨跌停不可成交、无 ST 过滤、滑点按固定值。
5. **无市场态势闸**:斐波策略震荡市连续小亏(自认),缺指数级 regime 过滤来跳过烂市。
6. **无实盘闭环**:只有信号,没有持仓台账、成交模拟、每日盈亏对账、风控熔断。
7. **"AI 决策模块"名实不符**:目前 AI 只在对话,没进交易决策环。

## 二、改造路线图(按「收益/风险杠杆」排序,分四阶段)

### 阶段 1 · 立刻提升收益质量(本周,纯代码/离线可验证)

- **[P1-A] 市场态势闸 `regime.py`** ⭐最高杠杆
  指数级 regime(沪深300 在 MA200 之上 / MA50 斜率 / 宽度)→ 输出 0/0.5/1 仓位敞口乘子。
  风险关时**跳过新进场或减半仓位**。趋势策略最经典的"月度一致性"提升手段,直接砍掉震荡市回撤。
  *预期:回撤从 -4.2% 进一步压低,坏月份变少 → 月度分布右移。*

- **[P1-B] 样本外 walk-forward 验证**
  把 `grid_search` 改成滚动窗口:前 N 月选参 → 后 M 月用该参跑,拼接 OOS 权益曲线。
  报告 **OOS** 而非样本内指标。这是"工业级可信度"的硬门槛(宁可数字变小,要真实)。

- **[P1-C] A 股微观结构回测修正**
  T+1(进场次日才可卖)、涨停不可买/跌停不可卖(用 open≈high≈low 近似判定)、
  ST/退市股过滤、按成交额分档的滑点模型。让回测从"乐观"变"可落地"。

### 阶段 2 · 扩大 Alpha 来源(2–3 周)

- **[P2-A] 全市场宇宙 + 流动性/上市天数过滤**
  从 32 只 → 全 A 扫描(剔除 ST、上市<250 日、日均成交额<阈值、价格<阈值)。
  趋势候选供给上一个数量级 → 非对称大赢家命中数量级提升。配增量行情库支撑。

- **[P2-B] 增量行情数据库(DuckDB/parquet)**
  一次全量回补 + 每日增量,本地列存。回测/寻优不再反复打网络,速度 ×10,且 point-in-time。
  多源失败转移(新浪 → 东财 → 腾讯)。

- **[P2-C] 双引擎集成 + 资金分配器**
  斐波(趋势)+ 多因子(横截面)不再各跑,做成一个 allocator:regime 风险开→趋势为主,
  风险关→现金/低波为主。组合层统一权益与风控。

### 阶段 3 · 实盘运维闭环(3–4 周)

- **[P3-A] 纸上交易引擎 `paper_trade.py`**
  每日读信号 → 维护真实持仓台账(JSON/SQLite)→ 次日开盘按规则模拟成交 → 逐日盯市盈亏。
  与回测对账(live-vs-backtest tracking error 监控)。
- **[P3-B] 风控与熔断**:单日最大回撤熔断、单票/单板块敞口上限、连续止损降杠杆、相关性约束。
- **[P3-C] 可观测性**:结构化日志 + 每日运行健康检查 + 异常告警(数据缺失/信号异常/偏离回测)。

### 阶段 4 · AI 决策模块(名副其实)

- **[P4-A] LLM 催化剂/风险否决层**:对候选票用 Claude 核查近期公告/财报/事件,给"催化剂分"
  与"红旗否决"(商誉爆雷、减持、问询函)。只做**否决与加权**,不做凭空选股(防幻觉)。
- **[P4-B] 元策略择时**:用 AI 在 regime 边界做"加/减/平"的可解释决策,留人工确认闸。
- **[P4-C] 收益归因 → 迭代**:每月自动跑归因(选股 vs 择时 vs 仓位),喂回参数迭代。

## 三、验收标准(工业级"做完"的定义)

1. 所有回测指标为 **样本外(OOS)**,且含 T+1/涨跌停/滑点修正。
2. 关键路径有单元测试(指标计算、无未来函数、仓位、止损)。
3. 纸上交易跑满 1 个月,live-vs-backtest 跟踪误差在阈值内。
4. 风控熔断、数据缺失、信号异常均有告警且可恢复。
5. 月度收益分布:目标 **月均 >2.5%、月胜率 >55%、单月 ≥8% 频率 >25%、最大回撤 <15%**。
